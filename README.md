# qt-web-bridge

This is primarily a fuck around with the new [qtbridge-rust](https://github.com/qt/qtbridge-rust).

It has some tauri style commands and messaging between rust and javascript (including `invoke`, `emit`, 
`listen` and `once` handling).

No idea if this is any good, or whether it'll ever be used elsewhere, but might be an interesting jump off
point for other projects.
